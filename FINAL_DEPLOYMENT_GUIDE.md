28. Turn **Enable Sign in with Google** on.
29. Paste the Google **Client ID** into **Client ID**.
30. Paste the Google **Client Secret** into **Client Secret**.
31. Click **Save**.
32. Click **Storage** in the left sidebar.
33. Click **New bucket**.
34. Set **Bucket name** to:

```text
offer-documents
```

35. Turn **Public bucket** on. The current app uses `getPublicUrl()` for uploaded offer PDFs.
36. Click **Create bucket**.
37. Click **SQL Editor** in the left sidebar.
38. Click **New query**.
39. Paste this SQL:

```sql
create policy "Offer document uploads are owned by the authenticated user"
on storage.objects
for insert
to authenticated
with check (
  bucket_id = 'offer-documents'
  and (storage.foldername(name))[1] = auth.uid()::text
);

create policy "Authenticated users can read their offer documents"
on storage.objects
for select
to authenticated
using (
  bucket_id = 'offer-documents'
  and (storage.foldername(name))[1] = auth.uid()::text
);

create policy "Authenticated users can update their offer documents"
on storage.objects
for update
to authenticated
using (
  bucket_id = 'offer-documents'
  and (storage.foldername(name))[1] = auth.uid()::text
)
with check (
  bucket_id = 'offer-documents'
  and (storage.foldername(name))[1] = auth.uid()::text
);

create policy "Authenticated users can delete their offer documents"
on storage.objects
for delete
to authenticated
using (
  bucket_id = 'offer-documents'
  and (storage.foldername(name))[1] = auth.uid()::text
);
```

40. Click **Run**.

## Production Compatibility Notes

- `postinstall` runs `prisma generate`, so Vercel builds use a fresh Prisma Client.
- `typecheck` runs `next typegen` before `tsc --noEmit`, so generated Next route types are present.
- Demo data lazy-loads Prisma only in the database seeding helper, so the public demo can render static seeded data without initializing Prisma.
- Stripe webhook handling supports:
  - `checkout.session.completed`
  - `customer.subscription.created`
  - `customer.subscription.updated`
  - `customer.subscription.deleted`
  - `invoice.payment_failed`
- Supabase Google OAuth uses Supabase as the Google redirect URI and the app route `/auth/callback` as the final application callback.
